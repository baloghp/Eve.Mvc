Simple Page with Master and Partial views

Defining partial view in the Markup
You define a partial view in the markup by using the "eve-partial" attribute. I have broken out the LandingPage to the following conponents:

Here is the markup for the main page: 
<!-- Header -->
<a name="about"></a>
<div class="intro-header" eve-partial="eve-Assets.Views.Sample.MasterAndPartials.IntroHeader.html">
 
</div>
<!-- /.intro-header -->
<div eve-partial="eve-Assets.Views.Sample.MasterAndPartials.Contents.html" eve-renderinstead> </div>
 
<a name="contact"></a>
<div class="banner" eve-partial="eve-Assets.Views.Sample.MasterAndPartials.Banner.html">
 
</div>
<!-- /.banner -->

The "contents" partial view contains references tot he individual content views: 
<!-- Page Content -->
<a name="services"></a>
<div class="content-section-a" eve-partial="eve-Assets.Views.Sample.MasterAndPartials.Content1.html">
 
</div>
<!-- /.content-section-a -->
<div class="content-section-b" eve-partial="eve-Assets.Views.Sample.MasterAndPartials.Content2.html">
 
</div>
<!-- /.content-section-b -->
<div class="content-section-a" eve-partial="eve-Assets.Views.Sample.MasterAndPartials.Content3.html">
 
</div>
<!-- /.content-section-a -->

Code for partial views
You can as before define code for the partial views:

As you can see you do not always have to provide code for each partial view. Partial views are evaluated after the master page is evaluated. But before the code for the page's view is run. Thus you can also access html generated by the partial views in the code of your page view. 
   [MasterView("eve-Assets.Views.Sample.SimpleMaster.LandingMaster.html")]
 [EmbeddedView("eve-Assets.Views.Sample.MasterAndPartials.LandingPage.html")]
 public class PageView : EmbeddedView
 {
     public override void ProcessView(System.Web.Mvc.ViewContext viewContext)
     {
         var nodes = this.HtmlDocument
                         .Document
                         .DocumentNode
                         .SelectNodes("//h2");
         foreach (var item in nodes)
         {
             item.InnerHtml = "Partial view content changed by page view";
         }
 
     }

Model for partial views
You can define the Model for the partial views by defining a databinding path for in the eve-partialmodel attribute. If you do not define this,the partial view will inherit the calling view's Model. (If you are using typed view take special note of this.) 
<a name="about"></a>
<div class="intro-header" eve-partial="eve-Assets.Views.Sample.DataBinding.IntroHeader.html"
     eve-partialmodel="Header">
 
</div>
<!-- /.intro-header -->
<div eve-partial="eve-Assets.Views.Sample.DataBinding.Contents.html"
     
     eve-renderinstead></div>
 
<a name="contact"></a>
<div class="banner" eve-partial="eve-Assets.Views.Sample.DataBinding.Banner.html" eve-partialmodel="Banner" >

In this example the headerand the banner will have their own child models, however the conent will inherit from the model of this view.
